# Contributing
If you want to contribute to this project, you have to fulfill these:
- Do not add extra comments to the code (comments kind of clutter the code, and i tend to remove comments), having a comment at the top of files that you want to license differently is okay though. If you are contributing code with an LLM, tell it to not include any comments.
- Always test your code before sending your patches, so you will know what to fix before sending patches

Tips:
- If you are willing to contribute with the usage of LLM, consider using claude models (either claude sonnet or claude opus, claude opus is recommended) since those tend to write fine code without breaking much stuff

Development Setup
- x86_64-elf-gcc for building the OS, libc and anything in userprog
- Literally any text editor, IDE or whatever
